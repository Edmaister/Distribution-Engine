# Capability Gap Matrix

This matrix evaluates the current repository against the Distribution Layer as a Service target state. Current facts are based on repo files, migrations, services, routers, and tests. Target-state recommendations are listed as required enhancements.

Status values:

- Exists: the repo has a concrete backend/schema/service capability.
- Partial: the repo has meaningful pieces, but not enough for reusable DLaaS platform readiness.
- Missing: no first-class implementation was found in the repo.
- Unknown: cannot be confirmed from static repo inspection; runtime/live DB inspection is required.

## Highest-Risk Platform Gaps

1. No full SaaS account/user/seat/subscription layer yet.
2. No single canonical distribution outcome spine.
3. Campaign, referral, distribution opportunity, reward, commission, funding, fulfilment, and settlement states are powerful but fragmented.
4. Sponsor billing exists, but SaaS platform billing and usage metering are missing.
5. Public APIs exist in pieces, but are not yet packaged as stable DLaaS contracts.
6. White-label/embed should wait until tenant isolation, safe statuses, and API contracts are hardened.
